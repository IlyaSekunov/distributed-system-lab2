# Отчет по параллельной программе вычисления множества Мандельброта с использованием OpenMP

## 1. Введение

### 1.1. Описание задачи
Множество Мандельброта — это фрактальное множество на комплексной плоскости, определяемое как множество точек c ∈ C, для которых последовательность:
```
z₀ = 0
zₙ₊₁ = zₙ² + c
```
остается ограниченной при n → ∞. Критерий принадлежности: если для какой-либо итерации |zₙ| ≥ 2, то точка не принадлежит множеству.

### 1.2. Цель работы
Разработка параллельной программы для генерации точек множества Мандельброта с использованием технологии OpenMP, исследование масштабируемости алгоритма на различных аппаратных архитектурах.

## 2. Реализация алгоритма

### 2.1. Основной алгоритм
Программа реализует стохастический метод построения множества:
1. Генерация случайных точек в области [-2.0, 1.0] × [-1.5, 1.5]
2. Проверка каждой точки на принадлежность множеству с использованием итерационного критерия
3. Сохранение координат принадлежащих точек в CSV-файл

### 2.2. Параллельная реализация
```c
#pragma omp parallel
{
    // Thread-local генерация и обработка точек
    #pragma omp for
    for (int i = 0; i < npoints; i++) {
        // Генерация случайной точки
        // Проверка принадлежности множеству Мандельброта
    }
    
    // Сбор результатов через критическую секцию
    #pragma omp critical
    {
        // Объединение локальных результатов

```

## 5. Анализ результатов

### 5.1. Масштабируемость по потокам
1. **Линейное ускорение до 4 потоков**: При 2 потоках достигается почти идеальное ускорение (91-100%), при 4 потоках — 70-84% эффективности.

2. **Дальнейшее увеличение потоков (6)**: Дает дополнительное ускорение, но эффективность снижается до 43-75%. Это объясняется:
   - Накладными расходами на управление потоками
   - Конкуренцией за ресурсы памяти
   - Ограничениями физических ядер (6 ядер)

### 5.2. Масштабируемость по объему задачи
1. **Больший объем → лучшее ускорение**: Для 10⁶ и 10⁷ точек ускорение выше, чем для 10⁴ и 10⁵ точек. Это объясняется:
   - Уменьшением относительной доли накладных расходов
   - Более эффективным использованием кэша процессора

2. **Стабильная пропускная способность**: Количество обрабатываемых точек в секунду остается относительно постоянным для каждого количества потоков, что свидетельствует о линейной зависимости времени от объема данных.

## 6. Выводы

### 6.1. Основные достижения
1. **Успешная параллелизация**: Программа демонстрирует значительное ускорение при использовании нескольких потоков.

2. **Максимальное ускорение**: 4.51 достигнуто при 6 потоках для 10⁶ точек, что составляет 75% от теоретического максимума.

3. **Линейная зависимость**: Время выполнения линейно зависит от количества точек, что соответствует ожиданиям для данного алгоритма.

### 6.2. Оптимальные параметры
Для тестовой системы оптимальными являются:
- **Количество потоков**: 4-6 (в зависимости от объема данных)
- **Минимальный объем данных**: 10⁵ точек для эффективной параллелизации
